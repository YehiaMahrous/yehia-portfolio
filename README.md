## **Project Overview**

This repository contains our final class project: a clean, professional, and accessible single-page web portfolio. It is fully responsive, utilizing semantic HTML and a utility-class approach in vanilla CSS to match modern web standards.

In addition to the raw application, we have provided documentation covering the wireframe/planning stage and a fast-track beginner manual outlining the build process.

## **Inventory of Generated Files**

The following files have been generated as part of this assignment:

- **`index.html`**
  - The core HTML document containing the structure, semantic sections (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`), and injected content tailored to a Data Science & AI Major.
- **`index.css`**
  - The styling rules, featuring CSS custom properties for theming, utility classes mimicking modern CSS frameworks (for tight layout control), hover animations, responsive Media Queries, and a pure CSS-based mobile navigation hack.
- **`wireframe.md`**
  - Step 1: The initial ascii-based layout wireframe, including the section-by-section content plan, planned colour palette, and accessibility considerations.
- **`manual.md`**
  - Step 3: The "Fast-Track" Instructional Manual aimed at complete beginners or career-changers entering the tech field. It uses a mentoring tone to break down how the HTML and CSS were written chronologically.
- **`README.md`**
  - Step 4: This active document; the final submission cover sheet containing the file inventory.

## **Accessibility & Responsiveness Highlights**

- **WCAG Compliance**: Passed colour contrast checkpoints (very dark backgrounds with `slate-100` and `slate-300` text).
- **ARIA Integration**: Used `aria-hidden` and `aria-label` correctly throughout the mobile navigation elements.
- **Device Support**: Media queries added to handle layout stacking on screens < 768px (like the navigation wrapping into a "hamburger" style menu dropdown).
